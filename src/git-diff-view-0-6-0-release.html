After 2 years of no updates, version 0.6.0 is here.

New features:

  • New UI for staged files.
  • New UI for diff view.
  • "Discard Changes" is now shown in a small popup menu, instead of a distracting modal.
  • Improvement to the logic responsible to restore the old window layout.

Breaking Changes:

  • Support for Sublime Text 3 is dropped! Users who are on Sublime Text 3 will just continue to use GitDiffView 0.5.0.

Other Changes:

  • A complete rewrite of the plugin.
  • Switched to Python 3.8.
  • Added Mypy for type-checking.

While the new update should brings improvements, it doesn't mean that it is perfect.

The new git diff view is now able to highlight/visualize char that had been added/removed, but the code that is used to for that doesn't detect the changes how I would expect. Right now difflib.ndiff is used. PR that could improve this are welcome.
